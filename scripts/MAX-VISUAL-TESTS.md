# MAX Visual Human Tests

Пошаговый ручной тест для проверки UX в приложении MAX.

## 0) Подготовка (2 минуты)

1. Запустите бота: `npm run dev`.
2. В отдельном терминале выполните: `npm run human:test`.
3. Откройте бота в MAX.

---

## 1) User flow (обычный пользователь)

- [ ] Отправить `/start`
  - Ожидаемо: есть акцент на `/guide` + inline-кнопки.
- [ ] Нажать `Инструкция для пользователя`
  - Ожидаемо: короткая 3-шаговая инструкция.
- [ ] Нажать `FAQ`
  - Ожидаемо: блок Q/A без ошибок.
- [ ] Нажать `Что дальше`
  - Ожидаемо: последовательный сценарий действий.
- [ ] Выполнить `/status`
  - Ожидаемо: summary статуса и роли.

---

## 2) Admin flow (owner/admin)

- [ ] Нажать `Открыть админку` в `/help`
  - Ожидаемо:
    - либо открывается ссылка (если `ADMIN_PANEL_URL` публичный `https`);
    - либо понятное объяснение про локальный URL.
- [ ] Выполнить `/wizard`
  - Ожидаемо: кнопки шагов 1..5.
- [ ] Нажать `1) Создать демо`
  - Ожидаемо: создан contest ID + подсказка следующего шага.
- [ ] Нажать `2) Опубликовать сюда`
  - Ожидаемо: в текущем чате появляется пост с кнопкой `Участвовать`.
- [ ] Нажать `3) Статус`
  - Ожидаемо: contest status + counters.

---

## 3) Draw/proof flow

- [ ] Нажать кнопку `Участвовать` в опубликованном посте.
  - Ожидаемо: callback-уведомление об участии.
- [ ] Нажать `4) Draw`
  - Ожидаемо: либо draw, либо корректное объяснение причины (например нет участников).
- [ ] Нажать `5) Proof`
  - Ожидаемо: seed + winners/статус.
- [ ] Выполнить `/proof contest_id`
  - Ожидаемо: совпадает с wizard proof.

---

## 4) Негативные сценарии

- [ ] `/draw` от пользователя без прав
  - Ожидаемо: `adminOnly`.
- [ ] `/join` с несуществующим contest_id
  - Ожидаемо: `contestNotFound`.
- [ ] Быстро 2-3 раза подряд нажать join/draw
  - Ожидаемо: защита cooldown с понятным текстом.

---

## 5) UI acceptance criteria

- [ ] Нет “битых” кнопок.
- [ ] Нет технических stack trace в ответах.
- [ ] Тексты консистентны по тону.
- [ ] Ошибки объясняют, что делать дальше.
